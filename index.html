Problem Description:

Quebes is an asset-based Infrastructure Provider that operates in 18 countries. It provides assets like Laptops, Desktops, VoIPâ€™s etc. for lease. Company wants to ease its process of registering new assets for efficient management. You will be helping the company to achieve its goal to ease its process for this business requirement.

Details of Backend REST API service (Implemented)-

  • Adding an Asset – This service adds an Asset by sending POST request at /assetDetails. The asset gets assigned to a asset Id.

    { 'message': 'Asset is created with assetId 1001' }

    •  

  • Returning details of an asset through id: This service returns a JSON response containing details of particular asset id by sending GET request at /assetDetails/<id>

    { 'assetId': 1001, 'assetName': 'Dell Inspiron', 'assetCategory': 'Laptop', 'assetDescription': '10th Gen Intel Core Processor, 8GB DDR4 RAM', 'dateOfPurchase': '10-2-2020', 'assetCost': 50000 }

  • In case asset id is not found

    { 'message': 'Asset not found !' }

    •  

    Note: The ' assetCost ' is in Indian Rupee

     

  • Updating details of an Asset through id: This service updates asset details by sending PATCH request at /assetDetails/<id>

    Note: Patch replaces only specified fields.

    Example: If client sends the PATCH request at http://localhost:3020/assetDetails/1001 with data {'assetName':'Dell'} then the output will be

    { 'message': `Asset with asset Id 1006 is updated successfully` }

    •
